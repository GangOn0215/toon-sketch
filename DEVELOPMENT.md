본 문서는 Toon Sketch 프로젝트의 주요 개발 진행 상황과 아키텍처 결정을 기록하는 로그입니다.

## 📅 프로젝트 시작: 2026-03-04

---

## 🚀 주요 마일스톤 (2026-03-04)

### 1. AI 연동 및 백엔드 구축
- **FAL AI (Flux.1) 통합**: `@fal-ai/client`를 사용하여 고성능 이미지 생성 엔진 연동.
- **API 라우트 설계**: 캐릭터 시트 생성을 위한 `/api/generate` 엔드포인트 구현.
- **프롬프트 엔진 최적화**: 한국어 사용자 선택지를 AI가 최적으로 이해할 수 있는 상세 영문 묘사(Descriptive English)로 변환하는 매핑 시스템 구축.

### 2. 캐릭터 빌더 워크스페이스 (UI/UX)
- **Essential/Detailed 옵션 분리**: 성별, 연령대, 의상, 화풍 등 핵심 설정은 상단에 배치하고 세부 설정은 아코디언 형태로 숨겨 사용자 피로도 감소.
- **실시간 캐릭터 소환**: 사용자의 선택에 따라 캐릭터 시트(전/측/후면)를 실시간으로 생성하고 확인하는 워크스페이스 구축.
- **캐릭터 고정(Seed) 기능**: 특정 캐릭터의 외형을 유지하며 옵션을 조정할 수 있는 Seed 고정 기능 구현.

### 3. 개발 생산성 및 안정성 강화
- **프롬프트 모듈화**: 서버와 클라이언트가 동일한 프롬프트 생성 로직을 공유하도록 `prompt-maps.ts`로 코드 분리.
- **TypeScript 안정성**: `any` 타입을 배제하고 `unknown` 및 타입 가드(Type Guard)를 도입하여 런타임 에러 방지.
- **개발자용 디버깅 도구**: 통신 실패 시에도 전송하려던 프롬프트를 확인할 수 있는 '실제 프롬프트 복사(Dev)' 기능 추가.

### 4. Gemini CLI 환경 설정
- **Custom Skills 도입**: 프로젝트 전용 Gemini CLI 스킬(`.gemini/skills/`)을 구축하여 코드 리뷰, 컨벤션 준수 커밋, 작업 로그 자동화 환경 조성.

## 🛠 기술 스택
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **AI Engine**: FAL AI (Flux.1 [dev])
- **Styling**: Vanilla CSS (Modern aesthetic)
- **Tools**: Gemini CLI, Custom Skills
