# 코드 리뷰 스킬

이 스킬은 정해진 체크리스트에 따라 코드를 분석하고 개선점을 제안합니다.

## 리뷰 체크리스트
1. **코딩 스타일 (PHP 5.6 / PSR-1 / PSR-2 기준)**
   - 들여쓰기: 4 spaces (탭 금지)
   - 클래스/메서드 중괄호는 새 줄에, 제어 구조 중괄호는 같은 줄에 위치하는지 확인
   - 명명 규칙: 클래스(PascalCase), 메서드(camelCase), 상수(UPPER_CASE)
   - PHP 5.6 호환성 (array() 문법 등) 및 PHPDoc 준수 여부
2. **보안 및 개인정보**: SQL Injection, XSS, CSRF 등 보안 취약점 확인
3. **성능**: N+1 Query 문제, 불필요한 루프, 메모리 효율성 분석
4. **에러 처리**: 적절한 예외 처리 및 엣지 케이스 대응 여부
5. **테스트**: 단위/통합 테스트의 필요성 및 커버리지 확인
6. **영향도 분석**: Breaking Change 유무 및 마이그레이션 필요성 검토

## 출력 형식
리뷰 결과는 항상 아래의 형식을 유지합니다:

- ✅ **잘한 점**: 칭찬할 만한 구현 방식이나 잘 지켜진 규칙
- ⚠️ **개선 제안**: 더 나은 방식이나 스타일 수정 제안 (가능하면 코드 패치 포함)
- ❌ **치명적 문제**: 반드시 수정해야 할 버그나 보안 이슈
  - **종합 점수**: (1~10점)
