# 업무일지 생성기

이 스킬은 지정된 날짜의 git 커밋 로그를 분석하여 체계적인 업무일지를 생성합니다.

## 워크플로우 (토큰 절약 우선)
1. 사용자가 날짜를 지정하면 해당 날짜의 커밋을 조회하고, 지정하지 않으면 오늘 날짜를 기준으로 합니다.
2. `git log --after="YYYY-MM-DD 00:00" --before="YYYY-MM-DD 23:59" --format="%h %s%n%b" --stat` 명령을 사용하여 커밋 메시지와 변경 파일 목록을 한 번에 가져옵니다.
3. 기본적으로 커밋 메시지와 `--stat` 정보만으로 일지를 작성합니다. (토큰 절약을 위해)
4. 메시지가 불명확하여 구체적인 수정 내용 파악이 꼭 필요한 경우에만 `git diff <hash>^..<hash> -- <파일경로> | head -80` 명령으로 핵심 파일만 제한적으로 확인합니다.
5. 내용을 **[추가]**와 **[수정]** 섹션으로 분류합니다.
6. 아래의 출력 형식을 엄격히 준수합니다.

## 토큰 절약 및 효율성 규칙
- `git diff` 전체를 읽지 말고, `--stat`과 커밋 메시지를 우선적으로 활용합니다.
- 대용량 파일(.sql 등)이나 자동 생성된 파일(.css, .map 등)은 diff를 조회하지 않습니다.
- 여러 커밋의 정보는 가급적 하나의 `git log` 명령으로 통합 조회합니다.

## 분류 기준
- **[ 추가 ]**: 새로운 파일 생성, 신규 기능 개발, 새로운 API/모델 추가 등.
- **[ 수정 ]**: 기존 코드 수정, 버그 수정, 리팩토링, 보안 패치 등.

## 출력 형식

```
**YYYY-MM-DD (요일) 업무일지**

**[ 추가 ]**

▷ 대분류 제목 (기능 단위 그룹화)
- 상세 설명 (사용자 관점에서 이해하기 쉽게)
- ...

**[ 수정 ]**

▷ 대분류 제목
- 상세 설명
- ...
```

## 작성 가이드라인
- **대분류(▷)**: 관련 있는 변경사항들을 하나의 테마로 묶습니다. (예: "로그인 API 보안 강화")
- **소분류(-)**: 구체적인 작업 내용을 서술하되, 기술 용어는 필요시 괄호로 설명합니다.
- 전체 분량은 섹션당 **6~10줄** 내외가 되도록 적절히 요약합니다.
- 요일은 한글(월~일)로 표기합니다.
