## 🔑 핵심 보안 강화 리스트 (Top 10)

1.  **결제 무결성**: 단건/정기 결제 모두 Toss API 응답값을 통한 **서버 측 실제 금액 대조** 완료.
2.  **경로 가드**: `middleware.ts`를 통해 비인가 사용자의 **비공개 경로 접근을 서버 수준에서 원천 차단**.
3.  **동시성 제어**: 다중 요청 시 크레딧이 마이너스가 되거나 중복 사용되지 않도록 **Atomic Update** 적용.
4.  **DoS 방어**: 유저별 **Rate Limiting** 및 생성 대기열 **타임아웃(Max Loops)** 설정.
5.  **스토리지 보호**: 이미지 URL 예측 공격을 막기 위한 **경로 난독화** 및 유저당 **저장 용량 제한**.
6.  **인증 보안**: `Authorization` 헤더 Bearer 검증 및 **Auth Callback Open Redirect** 완전 봉쇄.
7.  **데이터 정제**: SQL/Log 인젝션 방지를 위한 **에러 메시지 및 유저 입력값 필터링**.
8.  **서버 가용성**: 비정상 파일 다운로드 시 서버가 뻗지 않도록 **Memory Limit(OOM 방지)** 설정.
9.  **개인정보 보호**: 서버 로그에 카드번호, 이메일 등이 남지 않도록 **로깅 필드 마스킹**.
10. **통신 보안**: HTTPS 접속을 강제하고 중간자 공격을 방어하는 **HSTS 및 보안 헤더** 최적화.
