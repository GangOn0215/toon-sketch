## ✅ 구현 완료

### 1. 관리자 로그인 (`/admin/login`)
- 이메일 + 비밀번호 전용 (소셜 로그인 없음)
- **Brute Force 방어**: Upstash Redis 기반 Rate Limit (IP당 15분에 5회)
- 로그인 실패 시 남은 시도 횟수 표시
- 5회 초과 시 429 차단 + 재시도 가능 시간 안내

### 2. 관리자 대시보드 (`/admin`)
- 좌측 고정 사이드바 + 우측 콘텐츠 레이아웃
- 모바일: 햄버거 버튼 → 슬라이드 드로어
- **Stat 카드 4종**: 오늘 신규 가입 / 생성 수 / 매출 / 크레딧 소진
- **최근 가입자 테이블**: UID, 이메일, 플랜, 크레딧, 가입일시
- **생성 현황 바 차트**: 성공 / 대기 / 실패 비율

### 3. 접근 권한 제어
- **서버 측 (proxy.ts)**: 비로그인 → `/admin/login` 리다이렉트
- **서버 측 (proxy.ts)**: `profiles.role != 'admin'` → `/` 홈 리다이렉트
- **클라이언트 측 (page.tsx)**: 2차 role 확인
- 이미 로그인한 관리자가 `/admin/login` 접근 시 → `/admin` 자동 이동

## 🔒 보안 원칙

- 소셜 로그인(Google/Kakao/Naver) **사용 안 함** — 계정 분리 원칙
- 이메일 + 비밀번호 전용
- Brute Force 방어: IP당 15분 5회 제한 (Upstash Redis)
- 향후 강화 로드맵:
  1. TOTP 2FA (Google Authenticator)
  2. IP 화이트리스트

## 🗄️ Supabase 설정 필요 사항

```sql
-- profiles 테이블에 role 컬럼 추가
ALTER TABLE profiles ADD COLUMN role text DEFAULT 'user';

-- 관리자 계정 지정
UPDATE profiles SET role = 'admin' WHERE id = '관리자_user_id';
```
